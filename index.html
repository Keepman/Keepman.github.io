将hexo部署到github

发表于 2018-05-06 | 分类于 git
  1. 首先我们在github上新建一个项目仓库，必须取名为username.github.io

  2. 在hexo项目的根目录下的_config.yml中需要更改配置

    //需要注意的是这些冒号后面都需要跟上空格
deploy: 
  #我们是使用 git 来部署的。
  type: git
  # git 仓库的地址。
  repo: <仓库地址>
  # 分支名称。
  branch: master
  3. 上传项目至github
  4. 执行命令hexo clean && hexo g && hexo d

Coding和Github的比较

  • Github是国外免费的Git代码托管平台。利用Github Page服务可以免费创建一个静态网站。
  • Coding则是国内Git代码托管平台。国内首个Git代码托管平台GitCafe已被Coding收购。也提供page服务。

将项目托管到Coding

  1. 创建一个新项目
  2. 配置SSH公钥 公钥登录是为了解决每次登录服务器都要输入密码的问题，流行使用RSA加密方案
    1. 检查是否有SSH Keycd ~/.ssh如果没有.ssh目录则新建一个
    2. ssh-keygen -t rsa -C "your e-mail"注意：此处C为大写，your e-mail替换成自己的邮箱，引号不用加
    3. Enter passphrase (empty for no passphrase):<输入加密串>
      Enter same passphrase again:<再次输入加密串>这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容

hexo总结

发表于 2018-05-05 | 分类于 hexo框架
  1. 首先Hexo是基于Node.js的，所以我们需要安装Node.js
  2. 安装完成后我们运行命令npm install -g hexo-cli
  3. 运行命令hexo init <folder>hexo会帮我们自动创建一个文件夹
    • _config.yml：网站的配置信息
    • scaffolds：模版文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件
    • source：资源文件夹是存放用户资源的地方
    • themes：主题文件夹。Hexo会根据主题来生成静态页面
  4. cd <folder> 这一步千万不能忘，否则后面的都执行不下去了！！
  5. npm install
  6. hexo g 生成静态文件
  7. npm install hexo-server --save 在Hexo 3.0 把服务器独立成了个别模块，您必须先安装 hexo-server 才能使用
  8. hexo server 启动服务器。默认情况下，访问网址为： http://localhost:4000/

  • 自定义拦截器需要去实现HandlerInterceptor，并重写它的3个方法

    public class SecurityInterceptor implements HandlerInterceptor {

//业务处理器处理之前被调用，被拦截返回false，接下去的方法不执行，反之能正常到Controller层
public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        return false;
        }
    return true;
}

//在业务处理完成请求后，在DispatcherServlet向客户端返回响应前被调用 
public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
    System.out.println("进行中");
   }

//在DispatcherServlet完全处理完请求后被调用(可以在该方法进行一些资源的清理操作)
public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
    System.out.println("结束");
    }
}

上传文件到服务端

linux系统下用 ssh登录管理服务器，并用scp上传下载文件
登录ssh方法：

  1. ssh 主机

    ssh 120.78.155.0 （表示用root用户登录到服务器）

  2. ssh 用户@主机 

    ssh user1@120.78.155.0  （表示用user1登录到服务器）

  3. ssh 主机 -l 用户名 -p 端口 

    ssh120.78.155.0 -l user2 -p 8080 （表示user2用户用8080端口连接到192.168.1.155的服务器）
